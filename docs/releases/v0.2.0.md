# LinguaSpindle v0.2.0 — EPUB Technical Preview

LinguaSpindle v0.2.0 adds a bounded, structure-preserving EPUB 2/3 translation round trip to the
existing local-first translation orchestrator. It remains a single-instance Technical Preview:
there is no login or application user model, and remote access still requires an external private
network or access-controlling perimeter.

## Highlights

- Import common valid, unencrypted EPUB 2 and EPUB 3 packages; inspect package metadata,
  navigation, reading order, references, cover, XML/XHTML documents, and non-text resources.
- Translate documented visible text through stable document/XML-slot locators and the existing
  Provider, QA, retry, pause, resume, cancellation, and recovery boundaries.
- Rebuild from the immutable source archive, retain source text for failed or missing
  translations, update target-language declarations, reopen and independently validate the
  result, and byte-compare members that were not intentionally modified.
- Keep the accepted TXT novel and CBZ/image manga paths operational. Manga regression evidence
  uses the built-in deterministic Mock Adapter; it is not evidence of a real manga model run.
- Stream and bound source publication and Artifact downloads. Central guards cover upload bytes,
  ZIP member count, total and per-member expansion, compression ratio, path depth, unsafe names,
  encryption metadata, malformed packages, and ambiguous portable filenames.
- Reject imported raw or compressed content containing the active runtime Provider key, keep the
  key out of persisted configuration and diagnostics, and scan generated acceptance and Release
  material without recording the key value.
- Serve the no-login Web GUI, Typer CLI, and FastAPI HTTP API from one `ApplicationService` and
  durable `JobRunner`; interfaces do not execute Pipelines or third-party tools directly.

## Installation

Python 3.11 or newer is declared. Install the Wheel in a fresh virtual environment:

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install ./linguaspindle-0.2.0-py3-none-any.whl
linguaspindle --version
linguaspindle doctor
linguaspindle serve
```

Open <http://127.0.0.1:8765>. Mock TXT and EPUB operation needs no API key, Docker, GPU, model, or
external validator. The source archive can be installed in the same way by replacing the Wheel
path with `./linguaspindle-0.2.0.tar.gz`.

For Docker Compose, use the immutable `v0.2.0` source tag after publication:

```bash
git clone https://github.com/taoning0403/lingua-spindle.git
cd lingua-spindle
git checkout v0.2.0
cp .env.example .env
chmod 600 .env
docker compose build
docker compose up -d
docker compose ps
curl --fail --silent --show-error http://127.0.0.1:8765/health
```

Compose publishes only `127.0.0.1:8765` on the host, runs as UID/GID 10001 with a read-only root,
and stores all mutable state in the `linguaspindle-data` Volume. The core image does not include a
paid key, external manga stack, model, font, GPU runtime, browser, or external EPUB validator.

## Upgrade, backup, and rollback

v0.2.0 applies forward-only migration `0002_epub.sql` on first startup. Existing v0.1.0 Projects,
Sources, Jobs, Steps, Segments, QA findings, and Artifacts remain in the same data root.

Before upgrading, stop all writes and back up the **complete data root** as one consistent unit:
SQLite, WAL/SHM files when present, Artifacts, exports, logs, and cache. For Compose, stop the
service and archive the entire named Volume after confirming its actual project-prefixed name.
Never back up or restore only the database or only the Artifact directory.

There is no in-place schema downgrade. Rollback means stopping v0.2.0, restoring the complete
pre-upgrade data-root backup into an empty location or Volume, and then running v0.1.0. Do not run
v0.1.0 against a database that has already recorded migration 0002, and do not combine a restored
database with post-upgrade Artifact bytes. Detailed commands are in
[`docs/installation.md`](../installation.md) and [`docs/docker.md`](../docker.md).

## Technical Preview limits

- One host and one data root are the deployment and scheduling boundary. There is one SQLite
  database and an in-process worker, not a broker, distributed worker, or multi-host service.
- There is no registration, login, account, role, permission, tenant, ownership, quota, or
  collaboration model. Anyone with network reachability can operate the instance.
- Non-container startup binds to loopback by default, and supplied Compose publishes only to
  host loopback. Do not expose it directly to the public Internet.
- EPUB support targets common valid, unencrypted EPUB 2/3. It does not bypass DRM, repair every
  invalid publisher package, translate dynamically rendered JavaScript content, or add PDF,
  DOCX, MOBI, or AZW3.
- Built-in EPUB validation is structural and reference-oriented. Reader layout can still vary
  with publisher CSS, embedded fonts, namespace serialization, and the target reading system.
- The GUI provides progress, results, basic QA, controls, and downloads, not a professional
  sentence editor, CAT workflow, translation memory, or collaboration system.
- Manga streaming progress and immediate mid-image cancellation remain unchanged from v0.1.0.
