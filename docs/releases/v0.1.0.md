# LinguaSpindle v0.1.0 — Technical Preview

LinguaSpindle v0.1.0 is the first WSL2/Linux Technical Preview. It is a local-first,
single-instance translation orchestrator with no login or application user model.

## Highlights

- Standalone, no-login translation orchestration through a Web GUI, CLI, and HTTP API sharing one
  `ApplicationService` and durable Job core.
- TXT novel translation with Mock and OpenAI-compatible Providers, QA, and TXT/JSON exports.
- CBZ/image Mock flow plus a protocol-only Adapter for a separately operated
  `manga-image-translator` service.
- Persistent Jobs, Steps, Segments, logs, and immutable Artifacts with pause, resume, cancel,
  retry, restart recovery, and completed-work reuse.
- SQLite metadata plus an immutable local Artifact store under one data root.
- Loopback-first Docker Compose deployment with a non-root process, read-only root filesystem,
  health check, and persistent named Volume.

## Verified

- Ubuntu 26.04 LTS on WSL2 with the ordinary Linux Docker CLI.
- Docker Desktop Linux Engine 29.6.1 and Docker Compose 5.3.0.
- Non-root UID/GID 10001, read-only root filesystem, loopback-only host publish, and Volume
  persistence across restart, down/up without `-v`, build, and force-recreate.
- WSL Headless Chromium against the Docker GUI, with no login flow or browser-side Provider
  request.
- Mock TXT, expected Provider failure, Mock Manga, CBZ, TXT, and structured JSON flows.
- A minimal three-Segment real OpenAI-compatible Provider Job using `deepseek-v4-flash`.
- Exact runtime-key and Authorization/Bearer scans across Docker data, SQLite/WAL, logs, Artifacts,
  exports, screenshots, traces, downloads, and acceptance reports.
- Isolated wheel installation, packaged Web/migration resources, static checks, tests, and 82%
  branch-aware coverage.

## Limitations

- This is a Technical Preview, not a production-stable release.
- One host, one SQLite database, and one in-process worker are the deployment boundary.
- Supported source families are TXT novels and CBZ/images; EPUB is not included.
- The GUI provides result inspection and basic QA, not a professional translation editor.
- The real `manga-image-translator` model runtime remains unvalidated and is not bundled.
- Native Windows execution was not accepted in this release.
- Long documents, concurrency, rate-limit behavior, and long-term Provider stability remain
  unvalidated.
- Operators must independently verify licenses for external models, fonts, and other assets.
- The supplied deployment binds only to loopback. Remote access requires an external private
  network or access-controlling perimeter.

## Security

- LinguaSpindle has no user, login, role, permission, tenant, or ownership system.
- Provider API keys come only from the runtime environment and are not persisted.
- Provider secrets are excluded from configuration views, Job snapshots, logs, Artifacts,
  exports, fixtures, and Release assets.
- Mock flows require no paid key and remain the default automation path.
- Populated `.env` files, runtime databases, browser traces, and user inputs are not included in
  Git or this Release.

## Installation

Local Python installation from the source archive:

```bash
python3 -m venv .venv
.venv/bin/pip install ./linguaspindle-0.1.0.tar.gz
.venv/bin/linguaspindle doctor
.venv/bin/linguaspindle serve
```

Docker Compose from a source checkout:

```bash
cp .env.example .env
chmod 600 .env
docker compose build
docker compose up -d
docker compose ps
curl --fail --silent --show-error http://127.0.0.1:8765/health
```

Mock operation works without a real Provider key. If a real Provider is configured, keep its key
only in the protected runtime environment and never commit `.env`.
