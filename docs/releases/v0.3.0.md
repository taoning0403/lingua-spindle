# LinguaSpindle v0.3.0 release notes

v0.3.0 changes LinguaSpindle from a GUI-centered local application into a headless,
library-first translation orchestration engine. It removes caller-side product UI while retaining
TXT, EPUB 2/3, image, and CBZ translation as first-class capabilities.

These notes describe the v0.3.0 candidate surface. Exact commands, platforms, package artifacts,
checksums, and acceptance outcomes belong in the versioned `acceptance/v0.3.0/` archive and must
be reported there before publication.

## Highlights

- Side-effect-free `import linguaspindle`; no environment lookup, directory creation, SQLite
  access, or worker startup occurs on import.
- Stable typed Python APIs for document/manga inspection, selected translation, reconstruction,
  events, cancellation, errors, and versioned JSON-compatible results.
- TXT encoding/newline inspection, stable source locations, selected Segment translation, and
  UTF-8/LF reconstruction that preserves unselected source spans.
- Existing structure-preserving EPUB 2/3 inspection and reconstruction promoted into the pure
  public core with explicit `ArchiveLimits`.
- Pure image/CBZ manga inspection, page ordering, page-boundary cancellation, partial page
  results, Mock Manga Adapter, and separately installed real HTTP Adapter.
- Caller-supplied/manual translations can rebuild TXT or EPUB without calling a Provider.
- Provider and Manga Adapter remain distinct, minimal caller-implementable protocols.
- SQLite Project/Job/Step/Artifact persistence, CLI, OpenAI-compatible transport, real manga HTTP
  transport, and FastAPI server are isolated in optional dependency groups.
- The optional local runtime retains v0.2.0 data through additive migration
  `0003_headless_core.sql` and never starts a worker implicitly.
- The headless API exposes stable Segment listing, exact explicit selected translation (including
  an empty no-op), and Provider-free TXT/EPUB rebuild from a caller translation mapping.

## Public Python surface

Novel operations:

```text
inspect_document
extract_segments
translate_segments
rebuild_document
translate_document
```

Manga operations:

```text
inspect_manga
extract_manga_pages
translate_manga
build_manga_output
```

The public contracts include `DocumentManifest`, `Segment`, `SegmentLocator`, `MangaManifest`,
`MangaPage`, `TranslationProvider`, `TranslationRequest`, `TranslationRecord`,
`TranslationBatchResult`, `MangaTranslationAdapter`, `MangaTranslationResult`,
`TranslationOptions`, `TranslationEvent`, `CancellationToken`, `BuildResult`, `LinguaError`, and
`ErrorCode`. See [Python library API](../library-api.md).

## Installation model

```bash
python -m pip install linguaspindle
python -m pip install 'linguaspindle[openai]'
python -m pip install 'linguaspindle[manga]'
python -m pip install 'linguaspindle[runtime]'
python -m pip install 'linguaspindle[cli]'
python -m pip install 'linguaspindle[server]'
python -m pip install 'linguaspindle[all]'
```

The default dependency set contains only TXT/EPUB/core support. It does not include FastAPI,
Uvicorn, Typer, SQLAlchemy, Pydantic, python-multipart, platformdirs, HTTPX, Playwright, a browser,
an external manga tool, models, fonts, or a GPU stack. Missing optional features produce a clear
installation-extra diagnostic.

## GUI removal

v0.3.0 removes:

- `src/linguaspindle/web/` and all static HTML/JavaScript/CSS package data;
- GUI routes, SPA fallback, GUI-specific text, and browser-only acceptance;
- Playwright and Node/browser verification from the v0.3.0 package and test contract.

No replacement reader or editor is included. The optional server exposes JSON/OpenAPI only; its
root is a small headless service descriptor. Callers own readers, proofreading, approval/revision
history, bookshelves, and other business state.

## Preserved translation behavior

- TXT retains encoding detection and structure-aware segmentation.
- EPUB retains common valid unencrypted EPUB 2/3 package checks, visible-text policy, stable XML
  slot locators, source-based rebuild, target-language updates, internal references, unchanged
  resources, output reopen/reinspection, and archive limits.
- Manga retains single-image and CBZ/ZIP inputs, natural page order, safe archive processing,
  translated images, raw Adapter results, partial pages, normalized errors, page-boundary
  cancellation, `manga_full_v1`, Mock Manga, and the existing `manga-image-translator` HTTP
  protocol.
- Mock implementations remain deterministic and offline. They are not evidence of a paid model or
  real manga OCR/inpainting/typesetting execution.

## Upgrade from v0.2.0

The default core needs no data directory. Users retaining the optional local runtime should back
up the complete stopped v0.2.0 data root, install the relevant extras, and allow migration 0003 to
run once. The migration adds a nullable stable Segment key and partial unique index; it does not
delete or rewrite existing novel/manga rows or Artifact payloads. See the
[migration guide](../migrations/v0.2-to-v0.3.md).

Rollback is restore-based. Do not run v0.2.0 against a schema already migrated to version 3.

## Security and trust boundary

- The permanent no-user/no-auth/no-tenant boundary remains.
- The optional server binds to loopback by default. It must not be exposed directly to the public
  Internet; use an explicit outer private network or access perimeter.
- Provider credentials are caller-injected at runtime. Only optional interface configuration may
  resolve environment variables; credentials are excluded from model serialization, events,
  errors, logs, Artifacts, and output.
- EPUB and CBZ inputs remain bounded and reject traversal, unsafe paths, duplicate/ambiguous
  names, unsupported compression, encryption/protection, and configured size/ratio excess.
- The real manga service remains operator-managed and separately licensed. No upstream source,
  model, font, container, or GPU runtime is distributed by LinguaSpindle.

## Known limits

- Novel input remains TXT and common valid unencrypted EPUB 2/3 only; no PDF, DOCX, MOBI, AZW3,
  DRM bypass, or broad invalid-publisher repair.
- Manga editing remains whole-page Adapter output; v0.3.0 adds no bubble/region editing protocol.
- The current real manga HTTP Adapter has no streaming internal progress or immediate mid-image
  cancellation. Cancellation is observed after the active page call returns or times out.
- The pure API is synchronous. Embedders choose their own thread/task/queue execution model.
- The optional runtime remains one host, one SQLite database, and one local Artifact store; it is
  not a distributed queue or general DAG system.
- No reader, proofreading UI, CAT editor, approval workflow, plugin marketplace, or user system is
  included.

## Acceptance and publication

The release decision must use the completed `acceptance/v0.3.0/` report, not these notes. Real
paid Provider execution, real `manga-image-translator` model execution, external `epubcheck`, and
platforms not available to the acceptance environment must be identified separately as optional
external tests rather than represented by mocks.

Development work does not push a tag, publish a GitHub Release, or rewrite the historical
`acceptance/v0.2.0/` record. Publication requires separate authorization after the v0.3.0 report
and checksums are reviewed.
